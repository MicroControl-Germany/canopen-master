Detailed Description

The COM Queue module was introduced to have the possibility to process all messages of event-driven services like PDO. It is therefore possible to evaluate the content not only of the last received PDO, but of all PDOs in the queue. During initialization, a queue can be attached to a CANpie buffer of a CANopen service in order to write the messages of this service into a queue (a ring buffer). The Queue module offers the following possibilities to manage messages:

  • To link a queue with a buffer and release it again
  • Empty Queue
  • Query current number of messages
  • Adding Messages to the Queue (automatically done in the COM Stack Manager)
  • Removing Messages from the Queue

The queue is implemented as a ring buffer. If this is full, the old messages are removed, so that the new ones can be written. The size of the ring buffer is defined via the define #COM_QUEUE_LEN_MAX. The maximal number of queues is limited by the #COM_QUEUE_MAX define.
